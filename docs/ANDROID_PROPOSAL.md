# NexuSSH Android — план адаптации

Версия документа: draft 2026-05-30.
Цель: перенести NexuSSH на телефон с тем же «вау» как на десктопе. Фокус — стабильность + синхронизация. Без встроенного VPN, без SFTP в MVP.

## 1. Подход в целом

**Стек: Tauri 2 Android target.** Переиспользуем frontend (`src/` + xterm.js) и Rust-бэкенд (`russh`, vault, sync). Под телефон — отдельный mobile-layout (conditional UI в той же сборке), потому что десктопный sidebar + split + лента вкладок на 6-дюймовом экране не работают.

**Sync** — твой существующий age-зашифрованный файл, синкается чем пользователь поставит (Syncthing/Nextcloud/Dropbox). Логика идентична десктопу.

**Дистрибуция** — APK через GitHub Releases (как сейчас на десктопе), Play Store позже.

## 2. Главный экран

```
┌──────────────────────────────────┐
│ ☰   srv-prod-1 ▾   ●          ⋮  │  top bar 28px
├──────────────────────────────────┤
│                                  │
│  $ tail -f /var/log/syslog       │
│  May 30 22:01 srv-prod-1 kern..  │
│  ...                             │
│                                  │
│  (терминал на весь экран)        │
│                                  │
├──────────────────────────────────┤
│ [↑hist][Esc][Tab][Ctrl][^C][⏎]   │  smart bar — лента 1
│ [📋snip ▾][🔑pwd][↑][↓][←][→][⋯] │  smart bar — лента 2
├──────────────────────────────────┤
│      СИСТЕМНАЯ КЛАВИАТУРА        │
└──────────────────────────────────┘
```

- **☰** — drawer с хостами (наш sidebar в виде слайдера слева).
- **Центр top-bar** = имя активной сессии + статус-точка; тап = popover со списком открытых вкладок + «+ новая»; долгий тап = переименовать.
- **⋮** — меню сессии: рестарт, transcript, snippets, сменить тему сессии, закрыть.

Никакого split на телефоне — только на планшете в будущем.

## 3. Smart Key Bar — сердце мобильного UX

Это то место, где Termius выигрывает. Делаю две ленты, обе свайпаемы горизонтально (можно держать много кнопок).

### Лента 1 (постоянная)
- **↑ History** — посылает `↑` (история шелла). Долгий тап → overlay со списком последних команд, тапаешь — вставляется.
- **Esc, Tab, Ctrl, ^C, ⏎** — самое частое.

### Лента 2 (контекстная)
- **📋 Snippets ▾** — выпадашка с твоими кастомными командами на этот хост (или общими). Один тап = вставка. Редактируется в настройках хоста.
- **🔑 Password** — вставить пароль из vault для текущего хоста (полезно для `sudo`, или если хост настроен «спрашивать пароль каждый раз»). Защита: если vault залочен, сначала просит unlock.
- **↑↓←→** — стрелки кластером.
- **⋯ More** — раскрывает оверлей с F1-F12, PgUp/PgDn, Home/End, Insert/Delete, и любыми твоими кастомными.

### Sticky-модификаторы
Тап на Ctrl/Shift → залипает на следующее нажатие, второй тап = выкл, долгий тап = lock. Visual: жёлтое подсвечивание залипшего модификатора.

### Кастомизация
Всё дёргается в Настройки → Smart bar — порядок, что показывать, что прятать. Snippet'ы редактируются как маленькие json-объекты (имя + содержимое + опц. контекст «использовать только в bash»).

## 4. Жесты на терминале

- **Двупальцевый скролл** — scrollback.
- **Pinch** — размер шрифта налету (запоминается per-host).
- **Долгий тап** — стандартное меню «вставить / выделить слово / выделить всё / искать».
- **Тройной тап** — выделить строку.
- **Свайп от верха** на отключённой сессии = pull-to-reconnect.

## 5. Drawer (хосты)

Свайп от левого края или ☰. Внутри:
- Поиск.
- Папки (наш существующий tree).
- Per-host тап = info-карточка с большой кнопкой Connect.
- Долгий тап = меню (edit / duplicate / delete).
- Сверху быстрая полоска «Recent» — последние 5.
- Снизу — Settings и About.

## 6. Wow-фишки по фазам

### Фаза 1 (MVP, «стабильность + синхра»)
- Sync хостов из десктопного зашифрованного файла.
- Connect SSH (russh, та же логика).
- Smart bar с эссеншиалами (Esc, Tab, Ctrl, стрелки, Ctrl+C, history).
- Multi-session, переключение через top-bar popover.
- Themes (синкаются с десктопом).
- Vault unlock паролем.

### Фаза 2 (wow)
- **Snippets** — твои частые команды на тапе. С placeholder'ами типа `tail -f /var/log/{{service}}.log`.
- **Биометрика vault** — отпечаток/FaceID вместо мастер-пароля. Android Keystore + Argon2-ключ.
- **Quick-paste history** — долгий тап в терминале → последние 20 твоих вводов.
- **Output search** — лупа, ищет в scrollback с подсветкой.
- **PiP mode** — Android 8+, маленькое окно терминала поверх других приложений.
- **Auto-reconnect** на хэндофф Wi-Fi → 4G (foreground service).

### Фаза 3
- SFTP (адаптированный).
- Split на планшете.
- iOS port (когда у тех. поддержки активный dev-account).

## 7. Что НЕ переносим на мобилу

- Встроенный VPN (xray) — на телефонах у тебя уже свой VPN.
- Split-view панели (только одна сессия на экране, переключение через top-bar).
- Matrix Rain — по умолчанию выключен (батарея); опционально в настройках.
- Drag-and-drop хостов — на мобиле edit-режим (drag handle).
- SFTP в MVP.

## 7.5. Transcript + History на мобиле — ОБЯЗАТЕЛЬНО

На телефоне экран меньше, output убегает ещё быстрее → возможность пролистать историю сессии критически важна. Перенос:

- **Transcript overlay (live session)** — кнопка `↟ Scrollback` в верхнем баре сессии (или жест swipe-down с заголовка). Открывает full-screen overlay с replay-content. Закрытие = back-кнопка Android или X в верхнем углу.
- **History viewer (past sessions)** — пункт в drawer-е, список сессий, тап = открыть viewer. Так же full-screen.
- Архитектура — общая с десктопом (v1.0.1 fix: backend отдаёт cols/rows из cast header, replay ресайзит xterm в оригинальный размер, alt-screen обрабатывается нативно). Код 1:1 переиспользуем.
- На узком экране телефона запись с cols=120 не влезет — горизонтальный скролл (overflow-x: auto уже в фиксе) + pinch-to-zoom для уменьшения шрифта.

## 8. Theme / эстетика

Наш матричный стиль работает и на мобиле, но плотность ниже: дефолтный шрифт 14-16px (на десктопе 12-13). Темы те же, синкаются. Шрифты JetBrains Mono / Fira Code бандлятся в APK.

## 9. Transcript/History — статус фикса

**v1.0.1 (2026-05-30)** — десктоп ФИКС, commit 94c5169 в hipocrisle/nexussh:
- Backend `history_read_events` теперь возвращает `cols/rows` из cast header.
- Replay xterm ресайзит в `term.resize(originalCols, ...)` вместо `fit.fit()` — ESC sequences работают против правильного числа колонок.
- Alt-screen стрипанье убрано — xterm обрабатывает alt-screen нативно, main buffer scrollback остаётся чистым.
- Consecutive-line dedup убран.
- Cols пинятся при resize окна / fullscreen toggle.

CI собирает signed-релиз для авто-апдейтера. После апдейта Ctrl+Shift+↑ показывает корректный читаемый transcript. История в `Settings → History` — тоже.

## 10. Технические заметки реализации

- Tauri Android sdk 24+ (Android 7+). Подавляющее большинство девайсов.
- xterm.js работает на Android System WebView.
- Биометрика: `@tauri-apps/plugin-biometric` есть, но может потребовать ручной интеграции под мобильную Tauri 2.
- Storage Access Framework для выбора sync-папки (persistable URI).
- Foreground service для keep-alive SSH-сессий.
- Bottom safe area + WindowInsets handling — smart bar должен корректно подниматься над клавой.
